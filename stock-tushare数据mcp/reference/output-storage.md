# 数据输出规范

## 输出根目录

```
/Users/kirk/.openclaw/workspace/data/tushare/
```

**严禁写入 workspace 根目录。**

## 目录结构

按任务类型组织子目录：

```
data/tushare/
├── daily/          # 日线行情快照
├── financial/      # 财务报表数据
├── index/          # 指数数据
├── macro/          # 宏观经济数据
├── research/       # 研报、盈利预测
├── fund/           # 基金数据
├── hk/             # 港股数据
├── us/             # 美股数据
├── futures/        # 期货数据
├── bond/           # 债券/可转债
├── reports/        # 生成的分析报告
└── temp/           # 临时数据（可清理）
```

## 文件命名

```
{数据类型}_{标的}_{日期范围}.{格式}
```

示例：
- `daily_000001.SZ_20250101-20250228.csv`
- `income_600000.SH_2024Q1-2024Q4.csv`
- `index_000001.SH_20250201-20250228.csv`

## 输出格式

| 场景 | 格式 | 说明 |
|------|------|------|
| 临时查看 | 直接展示 | 小量数据直接在对话中展示 |
| 数据分析 | CSV | 大量数据存 CSV 供后续处理 |
| 报告输出 | Markdown | 分析结论用 Markdown 格式 |
| 模型输入 | CSV/JSON | 供财务模型使用 |

## 规则

1. 单次查询结果 < 20 行：直接在对话中展示，不存文件
2. 单次查询结果 >= 20 行：存文件，返回文件路径和摘要
3. 分析报告：存入 `reports/` 目录
4. 中间数据：存入 `temp/`，提醒用户可清理
