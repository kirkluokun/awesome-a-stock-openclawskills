这正是 Ontology 最该上的场景，董事长。                                                                          
 你这种“长讨论 → 初步结论”，用法是：过程不丢，结论可升级。                                                       
                                                                                                                 
 一套实操模板（最有用）                                                                                          
                                                                                                                 
 把一次长讨论拆成 4 类对象：                                                                                     
                                                                                                                 
 1. Question（问题）                                                                                             
 - 例：Q_柴发景气能持续到何时？                                                                                  
                                                                                                                 
 2. Evidence（证据）（可多条）                                                                                   
 - 会议纪要、研报、公告、数据点                                                                                  
                                                                                                                 
 3. Conclusion（阶段结论）                                                                                       
 - 标记为 status=tentative（暂定）                                                                               
 - 加上 confidence=0.62 这类置信度                                                                               
                                                                                                                 
 4. OpenRisk / NextCheck（待验证风险与下一检查点）                                                               
 - 例如：北美新增订单节奏、主机厂扩产兑现速度                                                                    
                                                                                                                 
 ────────────────────────────────────────────────────────────────────────────────                                
                                                                                                                 
 关系怎么连（核心）                                                                                              
                                                                                                                 
 - Evidence supports Conclusion                                                                                  
 - Evidence contradicts Conclusion（如果有反证）                                                                 
 - Conclusion answers Question                                                                                   
 - Risk threatens Conclusion                                                                                     
 - Task validates Conclusion（某日期复核）                                                                       
                                                                                                                 
 ────────────────────────────────────────────────────────────────────────────────                                
                                                                                                                 
 你以后就这么记（超短版）                                                                                        
                                                                                                                 
 每次讨论完，固定写 6 行：                                                                                       
                                                                                                                 
 1. 问题：XXX                                                                                                    
 2. 当前结论：XXX（暂定）                                                                                        
 3. 关键证据（最多3条）：A/B/C                                                                                   
 4. 反证或不确定点：XXX                                                                                          
 5. 置信度：0~1                                                                                                  
 6. 下次验证时间：YYYY-MM-DD                                                                                     
                                                                                                                 
 这 6 行一进 ontology，后面你就可以问：                                                                          
 - “这个结论现在还是成立吗？”                                                                                    
 - “上次结论的证据有没有被新信息推翻？”                                                                          
 - “哪些结论置信度在下降？”
